  ---
  双向同步方案总览

  正常流程（99% 场景）

  本机开发 → push GitHub → 脚本/cron 自动同步到 Gitee

  反向同步（83 服务器紧急补丁场景）

  用法：
  # 同步 Gitee 指定分支到 GitHub
  ./sync-gitee-to-github.sh 0g

  # 同步全部分支
  ./sync-gitee-to-github.sh

  冲突处理（Gitee 和 GitHub 同一分支都有新提交时）

  反向脚本 默认不用 --force，遇到 non-fast-forward 会自动报错停止，此时手动处理：

  # 1. 本地创建合并分支
  git checkout -b merge-gitee-fix gitee/0g

  # 2. rebase 到 GitHub 最新之上（把 Gitee 的补丁"搬"到 GitHub 最新之后）
  git rebase github/0g

  # 3. 如有冲突，解决后继续
  #    编辑冲突文件 → git add . → git rebase --continue

  # 4. 推送到 GitHub
  git push github merge-gitee-fix:0g

  # 5. 再正向同步回 Gitee（恢复镜像一致）
  ./sync-github-to-gitee.sh 0g

  # 6. 清理临时分支
  git checkout 0g && git branch -d merge-gitee-fix

  83 服务器上的推荐工作流

  如果 83 需要改代码并同步回来：

  # 83 上操作（从 Gitee 拉，改完提交到 Gitee 新分支）
  git fetch gitee 0g
  git checkout -b fix/hotfix-xxx gitee/0g
  # ... 改代码 ...
  git add . && git commit -m "hotfix: xxx"
  git push gitee fix/hotfix-xxx

  # 然后在本机或 43 执行反向同步（或用上面的 rebase 方式合入主分支）
  ./sync-gitee-to-github.sh fix/hotfix-xxx
