# Price Oracle Service Specification

## ADDED Requirements

### Requirement: 即时价格计算
系统必须计算流动性池的即时价格。

#### Scenario: 计算TRX/CHIP价格
- **WHEN** 系统查询TRX/CHIP价格
- **THEN** 价格 = CHIP储备量 / TRX储备量
- **AND** 必须考虑CHIP的6位小数精度

#### Scenario: 计算CHIP/TRX价格
- **WHEN** 系统查询CHIP/TRX价格
- **THEN** 价格 = TRX储备量 / CHIP储备量
- **AND** 价格单位为TRX per CHIP

### Requirement: 时间加权平均价格（TWAP）
系统必须计算时间加权平均价格，用于防范价格操纵。

#### Scenario: 计算TWAP
- **WHEN** 查询指定时间窗口的TWAP
- **THEN** 系统必须计算累积价格 / 时间间隔
- **AND** 累积价格 = Σ(价格 * 持续时间)

#### Scenario: 更新累积价格
- **WHEN** 每次储备量变化
- **THEN** 系统必须更新累积价格
- **AND** 记录当前区块时间戳

#### Scenario: TWAP时间窗口
- **WHEN** 查询TWAP
- **THEN** 系统必须支持不同时间窗口（如5分钟、1小时、24小时）
- **AND** 返回对应窗口的平均价格

### Requirement: 滑点计算
系统必须计算交易的滑点（价格影响）。

#### Scenario: 计算价格影响
- **WHEN** 用户输入交易数量
- **THEN** 价格影响 = 输入数量 / (源储备量 + 输入数量)
- **AND** 以百分比形式返回

#### Scenario: 高价格影响警告
- **WHEN** 价格影响 > 3%
- **THEN** 系统必须返回警告标志
- **AND** 建议用户调整交易数量或等待流动性增加

### Requirement: 交易输出计算
系统必须根据AMM算法计算交易输出。

#### Scenario: 计算TRX到CHIP输出
- **WHEN** 输入TRX数量
- **THEN** 输出CHIP = (输入TRX * 997 * CHIP储备) / (TRX储备 * 1000 + 输入TRX * 997)
- **AND** 扣除0.3%手续费

#### Scenario: 计算CHIP到TRX输出
- **WHEN** 输入CHIP数量
- **THEN** 输出TRX = (输入CHIP * 997 * TRX储备) / (CHIP储备 * 1000 + 输入CHIP * 997)
- **AND** 扣除0.3%手续费

#### Scenario: 最小输出计算
- **WHEN** 用户设置滑点容忍度
- **THEN** 最小输出 = 预期输出 * (1 - 滑点容忍度)
- **AND** 用于交易保护

### Requirement: 汇率缓存
系统必须缓存汇率数据以提升性能。

#### Scenario: 缓存汇率
- **WHEN** 计算汇率后
- **THEN** 系统必须缓存汇率和储备量
- **AND** 设置5秒过期时间

#### Scenario: 读取缓存汇率
- **WHEN** 查询汇率且缓存未过期
- **THEN** 系统必须返回缓存数据
- **AND** 避免重复计算和链上调用

#### Scenario: 缓存过期更新
- **WHEN** 缓存过期或储备量变化
- **THEN** 系统必须重新计算汇率
- **AND** 更新缓存

### Requirement: 历史价格查询
系统必须提供历史价格查询功能。

#### Scenario: 查询历史价格点
- **WHEN** 查询指定时间范围的价格历史
- **THEN** 系统必须返回该时间段的价格数据点
- **AND** 按时间升序排列

#### Scenario: 查询特定区块价格
- **WHEN** 查询特定区块高度的汇率
- **THEN** 系统必须从数据库读取该区块的储备量快照
- **AND** 计算当时的汇率

### Requirement: 价格波动计算
系统必须计算价格波动指标。

#### Scenario: 计算24小时价格变化
- **WHEN** 查询24小时价格变化
- **THEN** 系统必须计算 (当前价格 - 24小时前价格) / 24小时前价格 * 100%
- **AND** 返回百分比变化

#### Scenario: 计算价格波动率
- **WHEN** 查询价格波动率
- **THEN** 系统必须计算价格的标准差
- **AND** 用于风险评估

### Requirement: 流动性深度计算
系统必须计算流动性深度指标。

#### Scenario: 计算流动性深度
- **WHEN** 查询流动性深度
- **THEN** 系统必须计算在不同价格区间的流动性分布
- **AND** 用于深度图展示

#### Scenario: 计算流动性充足度
- **WHEN** 评估流动性是否充足
- **THEN** 系统必须计算储备量能否支持指定规模的交易
- **AND** 返回建议的交易上限

### Requirement: 手续费收益计算
系统必须计算流动性提供者的手续费收益。

#### Scenario: 计算LP收益率
- **WHEN** 查询LP收益率
- **THEN** 系统必须计算 (手续费累计 - 初始投资) / 初始投资 * 100%
- **AND** 年化收益率

#### Scenario: 计算累积手续费
- **WHEN** 查询累积手续费
- **THEN** 系统必须从历史交易记录中统计手续费总额
- **AND** 按LP占比分配

### Requirement: 无偿损失计算
系统必须计算流动性提供者的无偿损失。

#### Scenario: 计算无偿损失
- **WHEN** 价格发生偏离
- **THEN** 系统必须计算 LP价值 vs 简单持有价值的差异
- **AND** 显示无偿损失金额和百分比

#### Scenario: 无偿损失警告
- **WHEN** 用户添加流动性前
- **THEN** 系统必须显示潜在无偿损失风险
- **AND** 提供历史无偿损失数据参考

### Requirement: 价格预言机API
系统必须提供价格预言机API接口。

#### Scenario: API返回完整价格信息
- **WHEN** 调用价格查询API
- **THEN** 系统必须返回即时价格、TWAP、24小时变化等信息
- **AND** 响应格式为JSON

#### Scenario: API响应时间
- **WHEN** 调用价格API
- **THEN** 响应时间必须 < 100ms
- **AND** 优先使用缓存数据

### Requirement: 价格精度处理
系统必须正确处理价格精度问题。

#### Scenario: 价格小数位处理
- **WHEN** 显示价格给用户
- **THEN** 系统必须保留合适的小数位数
- **AND** 大价格保留2位小数，小价格保留6-8位

#### Scenario: 价格舍入
- **WHEN** 计算价格涉及除法
- **THEN** 系统必须使用安全的舍入算法
- **AND** 避免精度损失影响交易

### Requirement: 多交易对价格（扩展性）
系统应支持未来多交易对的价格查询。

#### Scenario: 查询多交易对价格
- **WHEN** 部署多个流动性池后
- **THEN** 系统必须支持按交易对地址查询价格
- **AND** 返回对应交易对的汇率

#### Scenario: 跨池价格计算
- **WHEN** 查询间接交易对价格（如通过TRX中转）
- **THEN** 系统必须计算多跳路径的综合价格
- **AND** 用于最优路由选择

### Requirement: 价格监控与告警
系统必须监控价格异常波动。

#### Scenario: 检测异常波动
- **WHEN** 价格在短时间内剧烈变化（如5分钟内变化 > 10%）
- **THEN** 系统必须记录告警日志
- **AND** 通知管理员

#### Scenario: 价格操纵检测
- **WHEN** 检测到可疑交易模式（如大量闪电贷交易）
- **THEN** 系统必须标记为可疑交易
- **AND** 触发安全审计

### Requirement: 价格数据备份
系统必须备份价格历史数据。

#### Scenario: 定期备份价格数据
- **WHEN** 每日定时任务执行
- **THEN** 系统必须备份前一天的聚合价格数据
- **AND** 保留至少90天的历史数据

#### Scenario: 数据恢复
- **WHEN** 数据丢失或损坏
- **THEN** 系统必须能从备份恢复价格数据
- **AND** 或从链上事件重新构建

### Requirement: 性能优化
系统必须优化价格计算性能。

#### Scenario: 批量价格查询
- **WHEN** 查询多个交易对价格
- **THEN** 系统必须使用批量查询减少数据库往返
- **AND** 并行计算提升性能

#### Scenario: 价格预计算
- **WHEN** 流动性池状态更新
- **THEN** 系统必须预计算常用价格指标
- **AND** 存储到缓存供快速查询
