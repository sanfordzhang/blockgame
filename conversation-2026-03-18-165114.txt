  在 leaveTableFor 方法中添加了抽水计算逻辑：

  1. 计算盈利：profit = finalStack - buyInAmount
  2. 只对盈利抽水：如果 finalStack > buyInAmount，才计算抽水
  3. 抽水计算：rake = (profit * session.rakeRateUsed) / 10000

  6. 发出事件：触发 GameSettled 事件记录抽水金额

  - 只对盈利部分抽水，亏损玩家不扣费
  - 使用 session.rakeRateUsed（玩家加入时的费率）保证公平
  - 返还给玩家的是扣除抽水后的净金额

  function vs event 的区别

  function（函数） = 在区块链上执行的操作，会改变状态
  event（事件） = 函数执行后发出的通知，方便外部监听

  玩家调用 joinTableFor → 区块链执行 → 状态改变 → [可选] 发出事件通知

  TRX 实际流向

  玩家钱包
    │
    │ deposit()
    ▼
  合约地址（TRX 锁在这里）
    │
    │ joinTableFor()  → balance → locked（合约内部记账）
    │ leaveTableFor() → locked → balance（合约内部记账）
    │                          → 抽水 → 服务器钱包
    │
    │ withdraw()
    ▼
  玩家钱包（TRX 取回）
